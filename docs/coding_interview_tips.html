Coding Skills Tips

  1. Use descriptive variable naming (example: is_value_positive instead of b for a boolean checking if a variable is positive)
  2. Use abstraction: separate portions of code that do specific tasks in other named ‘helper’ methods. Add the possibility to factorize the code and make it easier to read
  3. Use documentation (when needed for non trivial parts for example)
  4. Use descriptive code (example: the condition of a complicated if statement - with multiple and/or conditions - may be store in a named variable that make the condition easier to understand and be used after in the if statement)
  5. Use idiomatic coding style: use the correct way of writing a function and use built in functions (like map, enumerate, zip in Python - and also PEP8)


Problem Solving Tips

  1. Eliminate ambiguity: ask clarifying questions (like in real world problems) about assumptions to make or edge cases
  2. Take time to discover the problem (don’t jump directly into coding): show that you take time to understand the whole problem and think about alternative before diving into a first solution
  3. Discuss the tradeoffs (pros and cons) about style (clean implementation, readability), runtime, memory, … and compare with other solutions (for example: iterative vs recursive, dfs vs bfs)
  4. Implementing or tackling the problem in an organize manner: declare classes if needed, helper methods, factorie code, simplify it
  5. Test proactively: use a rigorous robust (edge cases, tricky things to think about, …)


Communication Tips

  1. Eliminating ambiguity: show that you gather information before diving in the code
  2. Voicing thought process: important to voice the different possible approaches or ideas in general
  3. Discussing tradeoffs: time vs memory vs readability
  4. Proactively asking for approval or help from interviewer:
  5. Do not over communicate, do not talk every second, do not discuss tradeoff for every line, etc.
